Hints and tips:

You can enter fractions (1/4 1/6 etc.) in the Quantity field for order editor.
When you use the green Squares to mark an item, it orders the same qty as last time.

In the case of sessions this shows the individual purchase quantity:

The two columns on the very right show the average sold per session, and the number of sessions that the remaining stock would therefore last for. 
If the number of sessions is less that 0.75 it is coloured red, if less than 1.5 it's pink.

You can see notes on each product by hovering the mouse over the left or right hand cells in each row.

Notes panel.

Hovering over the product code, or the notes icon () shows you a preview of the notes:

Clicking on the product code, or the notes icon, brings up an editable notes panel,
where you can record comments about this item for future reference.

The links across the bottom, allow you add text in the correct format to mark the item...
  • .. for order (with quantity)    - Use Add link
  • .. with a re-order quantity preset    - Use Remove link when already in order
  • .. as NO-Reorder    - Use NoReorder link
  • so that it refers to other items (e.g. substitutes)    - Use Also link
You can remove these attributes by simple deleting the "ORDER/ALSO" line from the notes. You can adjust the order quantity by editing the number in the "ORDER:" line.

Expanded view:

Here all sessions are show with 1 group of 4 cells per session.

This view shows "virtual stocktakes" for days when there was in fact no stocktake.

Notes:

The notes that you can see on normal view, are editable in expanded view without bringing up the Notes Panel.

When you are done, click on the Save button beneath.
The Save button is normally greyed-out, but becomes usable when the notes are edited.

These notes are now visible at the till. You can use them to explain ordering policy

Ordering:

Products can be flagged for ordering by inserting an appropriate line in the notes for the product.

ORDER:GH064,1

The Add link in the "Quicklinks:" section does this for you, The Remove link takes it away.
Remember:You still need to save the notes afterwards.

See also "green Squares" below.

No Re-Order:

Products can be flagged so they never appear normally in the analysis,
by inserting a different appropriate line in the notes for the product:

NOREORDER:CV253,1

The "NoReorder" link below the notes does this for you.
Remember:You still need to save the notes afterwards.

Filters:

There are 6 levels of filter, together with a textual filter. This cuts down the number of products on-screen and allows you to think about each in turn:

  • All levels - Shows all products EXCEPT NOREORDER products.
  • 0 - 0.75 - Shows products that very probably need ordering.
  • 0.75 0 1.5 - Shows products that might need ordering.
  • 1.5 - 2.5, 2.5 - 999 - etc,
  • Infinity - Shows products that have NOT SOLD AT ALL during the averaging period
  • Needs S/Tk - Products that haven't been stocktaken in the last averaging period.
  • No-Reorder Lines - Shows products that we've decided not to re-order. These are not shown in any other filter setting.
  • Current Order - Shows products currently flagged for order : ().
  • In Last order column - Shows products that have been ordered/delivered for the last session.

The text filter can be used to enter codes: "FX503" or "oats"

There are 2 special text filter settings:
- "order" shows you products flagged for ordering,
- "anomaly" shows you products flagged with ! in their stocktake columns because of "evaporation", or unexpected loss of stock.

Max sold

There is a max sold column which shows the max amount of the item that is sold in any one session. Hovering over it shows the date, so you could investigate further.:

if there is a large blue exclamation mark : , then that means that the max. qty is greater than the case size.

Uploading Copy invoices:

Suma sends us a copy invoice from Suma Wholefoods (<support@suma.coop>).
This is a very good starting point for entering a delivery (aka order) into the analysis.
If in fact they don't deliver exactly to the copy invoice, then you can edit individual quantities once the column has been created.
But the copy invoice upload is a quick way of creating the column and getting started.

To upload an invoice:

  • Open the email for the main order in your usual email reader (Outlook Express works).
  • save the attachment (called comething like 16116.A16577.html) to somewhere on your computer (temporarily)
  • go to the Analysis tool at ipswichripplefoodcooperative.co.uk/till/tillanalyse.pl
  • take the "Orders -> Copy invoice Upload" option:
       
  • use the "Choose File" or Browse button to find your temporary file.
  • set the "date to save as" be the ordering Tuesday (e.g.7/07/09) but the default (today) will probably be ok.
  • click "Save to database"
and you should have a new column with the order in it.
You can check this by going back to the Analysis tool and refreshing it
using the "In Last Order Column" filter option:
       

Green Squares

This is a quick(er) way to add items to the current order.

If you click on the "Sess remain" square, it will turn green thus:

You then have to click the refresh/Update button to actually add the tagged items to the current order

The same method cannot be used to remove items from the current order, Your have to go to the Notes for the product, and remove the following line from the notes:
  LASTORDER:XXXX,1
- where XXXX is the item code.

Creating stocktake columns:

This can be done with the Stocktake editor.
There's a link to this at the botton of the analysis screen.
It will offer to edit an existing stocktake, or create a new one.

Checking the SUMA shopping cart against order data

The "Compare shopping cart" tool lets you do 3 things:

- Preview the cart in alphabetical order of product code.
  Since the analysis tool orders its lists this way, this is a quick way to check against a paper list.

- Compare the cart with the current order set up in the analysis tool
  these are the items that have been set with the "Add" button in their notes field.

- Compare the cart with a composite of all personal orders since the last session.
  these are the items shown in the Personal order history composite pane

To use this tool -
- First go to the SUMA page displaying the shopping cart.
- Next save the current page to a temporary file on your computer (Firefox: File -> Save Page As..)
- Then use the browse button in the compare tool to browse to your saved cart

- Next use one of the buttons underneath to compare or preview it.

Ordering Cycle / Strategy

Ordering cycle:

Clear out previous order
Use 0-0.75 filter for obvious items.
Use 0.75-1.5 filter for not-so-obvious items.
Wishlist items.
Consider SeeAlso items.
Consider items for NoReorder.
Log in to SUMA
Copy order to SUMA basket
Check basket (& pricecheck)
Finalise Order
Upload copy invoice
Add new items to Analasys Tool
Choose stocktake tables
Print Stocktake sheets
Enter stocktake results

Edit / Fine tune

You can edit the following

  • Order qty (of current order)
  • Order qty (Historic)
  • Stocktake qty(Historic)

Reports

The tool records data files as it is used. You can see the following

  • Historic stock level (with contemporary prices) as csv file
  • Stocktake sheets
  • Stock levels for the Till to use (in javascript order)

Diaries

The Order diary can be used to record events / considerations
The Stocktake diary mostly records which tables were entered using the results forms,
but you can add extra comments if needs be.

Exclusions

Certain items with local order codes (non AANNN) are excluded, so you won't see any TOT codes.
